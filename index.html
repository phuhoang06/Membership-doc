§ 1 — Foundation

1. Tổng quan Kiến trúc

Tính năng Membership (B2C) được thiết kế độc lập hoàn toàn với hệ thống Pricing Plan (B2B) hiện tại để tránh xung đột logic.

  • Sử dụng Oracle Database cho mọi bảng dữ liệu.
  • Cơ chế Cronjob sử dụng ShedLock để chống chạy trùng lịch giữa các Pods.
  • Tích hợp Entity-driven Promotion cho việc tự động áp dụng giá Member (Không sử dụng Smart Sell/Triggered Addon).
  • Gửi Email qua bgs-email-service thông qua API nội bộ (Không gọi SendGrid trực tiếp).

Không tái sử dụng Pricing Plan: Vì B2B Subscription và B2C Membership có tính chất và luồng hoạt động hoàn toàn khác nhau. Việc ép dùng chung sẽ gây ra Technical Debt nghiêm trọng sau này.

§ 2 — Database

2. Oracle DB Schema

2.1. TB_MEMBERSHIP_PROGRAM (Cấu hình gói)

  • S_ID (VARCHAR2)
  • S_STORE_ID (VARCHAR2) - Liên kết với Store
  • S_NAME (VARCHAR2)
  • S_BILLING_CYCLE (VARCHAR2) - MONTHLY, YEARLY
  • S_PRICE (VARCHAR2) - Giá gói (thập phân)
  • S_STATE (VARCHAR2) - active, inactive
  • S_GLOBAL_DISCOUNT_TYPE (VARCHAR2) - PERCENTAGE, FIXED_AMOUNT
  • S_GLOBAL_DISCOUNT_VALUE (VARCHAR2)
  • C_CUSTOM_PRODUCT_DISCOUNTS (CLOB) - Yêu cầu thêm CHECK (C_CUSTOM_PRODUCT_DISCOUNTS IS JSON).
  • D_CREATE, D_UPDATE (TIMESTAMP)

2.2. TB_MEMBER (Hội viên)

  • S_ID (VARCHAR2)
  • S_USER_ID (VARCHAR2) - Liên kết với Buyer
  • S_PROGRAM_ID (VARCHAR2) - Liên kết với TB_MEMBERSHIP_PROGRAM
  • S_STATE (VARCHAR2) - active, on_hold, expired
  • N_AUTO_RENEW (NUMBER(1)) - 1 = true, 0 = false
  • D_NEXT_BILLING_DATE (TIMESTAMP)
  • S_PAYMENT_ACTION_URL (VARCHAR2) - URL xác thực 3DSecure của Stripe khi charge lỗi
  • C_CUSTOM_FIELDS_DATA (CLOB) - JSON lưu custom form
  • D_CREATE, D_UPDATE (TIMESTAMP)

2.3. TB_MEMBERSHIP_INVOICE (Lịch sử thanh toán)

  • S_ID (VARCHAR2)
  • S_MEMBER_ID (VARCHAR2) - Liên kết với Member
  • S_AMOUNT (VARCHAR2)
  • S_STATE (VARCHAR2) - paid, failed, pending, refunded
  • S_STRIPE_INVOICE_ID (VARCHAR2)
  • D_CREATE, D_UPDATE (TIMESTAMP)
§ 3 — Logic

3. Business Flows

Tích hợp Promotion Service

  1. Bổ sung type MEMBERSHIP_DISCOUNT vào bảng promotion type.
  2. Backend tự động insert 1 record Promotion ẩn (System-generated) map với Program ID khi Seller tạo gói.
  3. Tạo MembershipDiscountStrategyImpl extends PromotionStrategyAbs.
  4. Cart/Checkout (bgs-psp-buyer) check JWT, nếu user là ACTIVE member thì tự động append Promotion ID vào request.

Luồng Auto-Renewal & Dunning

  • Chạy tại bgs-billing-service với @Scheduled + @SchedulerLock.
  • Query charge: SELECT * FROM TB_MEMBER WHERE S_STATE='active' AND N_AUTO_RENEW=1 AND D_NEXT_BILLING_DATE <= SYSDATE.
  • Retry quá hạn (3 ngày, 5 ngày). Quá 7 ngày update S_STATE='on_hold'.

Luồng Xử lý Stripe Webhook & 3DSecure

Xử lý tại bgs-payment-service sau đó dispatch event.

  • Event invoice.payment_action_required: Bắn email yêu cầu xác thực 3DS. Cập nhật URL xác thực vào trường S_PAYMENT_ACTION_URL của bảng TB_MEMBER.
  • Event invoice.paid: Cập nhật D_NEXT_BILLING_DATE, xóa S_PAYMENT_ACTION_URL, và cập nhật status về active (nếu đang bị on_hold).
Admin API · bgs-psp-admin

Admin / Seller APIs

Các API dùng cho màn hình Dashboard của Seller. Header bắt buộc: Authorization: Bearer <Admin_JWT>.

A1 POST Create Membership Program
/ssp/api/v1/membership-programs

Purpose: Tạo một gói Membership mới. Khi gọi thành công, Backend tự động kích hoạt tiến trình tạo 1 Promotion ẩn trong DB.

Request Body

{
  "name": "VIP Burger Club",
  "billing_cycle": "MONTHLY",
  "price": "9.99",
  "global_discount_type": "PERCENTAGE",
  "global_discount_value": 15,
  "custom_product_discounts": {
    "prod_123": { "type": "FIXED_AMOUNT", "value": "5.00" }
  }
}
A2 GET Get Program Details
/ssp/api/v1/membership-programs/{id}

Purpose: Lấy chi tiết cấu hình của 1 chương trình Membership.

Params

NameInTypeRequired
idPathStringYes
A3 PUT Update Program
/ssp/api/v1/membership-programs/{id}

Purpose: Cập nhật thông tin gói (giá, tên, cấu hình giảm giá).

A4 GET List Members
/ssp/api/v1/members

Purpose: Lấy danh sách hội viên. Hỗ trợ phân trang và filter theo status.

Query Params

NameInTypeDescription
statusQueryStringACTIVE | ON_HOLD | EXPIRED
pageQueryIntegerDefault: 1
A5 POST Approve Member (Manual)
/ssp/api/v1/members/{id}/approve

Purpose: Duyệt tay cho hội viên (nếu setup yêu cầu approve tay thay vì auto-approve sau khi pay).

A6 POST Create Complementary Member
/ssp/api/v1/members/comp

Purpose: Tặng thẻ hội viên miễn phí (Comp) cho user. Không yêu cầu charge thẻ.

Buyer API · bgs-psp-buyer

Buyer / Storefront APIs

Các API dùng cho Storefront. Header bắt buộc: Authorization: Bearer <Buyer_JWT>.

B1 GET Get Membership Status
/psp-buyer/api/v1/membership/status

Purpose: Lấy trạng thái member hiện tại của user đang đăng nhập (để FE quyết định hiển thị giá VIP hay giá thường).

Response

{
  "code": 200,
  "data": {
    "is_member": true,
    "status": "ACTIVE",
    "next_billing_date": "2026-07-10T12:00:00Z",
    "program": {
      "name": "VIP Burger Club",
      "global_discount_value": 15
    }
  }
}
B2 POST Subscribe
/psp-buyer/api/v1/membership/subscribe

Purpose: Đăng ký gói mới. Trả về Checkout URL của Stripe để user nhập thẻ.

Request Body

{
  "program_id": "prog_091823",
  "custom_fields": {
    "t-shirt-size": "L"
  }
}
B3 PUT Cancel Auto-Renewal
/psp-buyer/api/v1/membership/cancel

Purpose: Khách chủ động tắt gia hạn. Đưa auto_renew về 0.

Lưu ý: Khi tắt gia hạn, trạng thái vẫn là ACTIVE cho đến ngày next_billing_date, sau đó mới chuyển sang EXPIRED.

B4 GET List Invoices
/psp-buyer/api/v1/membership/invoices

Purpose: Liệt kê lịch sử hóa đơn trừ tiền membership hàng tháng.

B5 PUT Update Payment Method
/psp-buyer/api/v1/membership/payment-method

Purpose: Đổi thông tin thẻ tín dụng mới. Dùng khi khách vào trạng thái ON_HOLD do thẻ cũ hết tiền/lỗi.
